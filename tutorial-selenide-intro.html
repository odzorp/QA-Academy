Selenide Fundamentals

Elegant Selenium Wrapper for Concise & Reliable Browser Automation

⏱️ Reading Time: 35 min
📊 Difficulty: Intermediate
🔧 Language: Java
📅 Updated: March 2026

Selenide is a powerful wrapper around Selenium WebDriver that simplifies browser automation in Java. Built to address common pain points in Selenium testing, Selenide provides a more intuitive API, automatic waiting, and built-in support for common testing scenarios. If you've struggled with flaky tests, complex waits, or verbose code in Selenium, Selenide offers an elegant solution.

What is Selenide?

Selenide is an open-source framework that wraps Selenium WebDriver to provide a cleaner, more reliable API for writing browser automation tests in Java. Created by Andrei Solntsev in 2011, Selenide has gained significant popularity in the Java testing community, particularly in Eastern Europe and Russia.

💎

Why Choose Selenide?

Selenide solves the most common Selenium headaches: flaky waits, stale element references, and verbose code. It uses a fluent API design that makes your tests readable and maintainable.

Key Features

⏰
Automatic Waiting
Selenide automatically waits for elements to appear, be clickable, or be visible. No more explicit waits or sleep statements.
📝
Fluent API
Concise, readable syntax that reads like natural language. Write less code with more clarity.
🔒
Automatic Screenshots
Built-in support for capturing screenshots on test failures automatically.
🐛
Clear Error Messages
Detailed error messages with screenshots included when tests fail, making debugging easier.
🌐
jQuery Support
Use jQuery-like selectors for element location, familiar to web developers.
⚡
Ajax Support
Built-in handling for dynamic web pages with Ajax calls and dynamic content loading.

Selenide vs Selenium

Understanding the differences between Selenide and raw Selenium helps you decide when to use each. Here's a detailed comparison:

Feature Selenium WebDriver Selenide
Waiting Manual explicit waits required Automatic waiting by default
Element Lookup Verbose findElement calls Fluent $() syntax
Stale Elements Must handle manually Automatic retry
Screenshots Manual implementation Automatic on failure
Configuration Multiple setup steps Single line initialization
Collections WebElement List Filtered SelenideElements
Language Java, Python, C#, etc. Java only

Setting Up Selenide

Getting started with Selenide is straightforward. You can add it to your project using either Maven or Gradle.

Maven Dependency

Add the following dependency to your pom.xml file:

<dependency> <groupId>com.codeborne</groupId> <artifactId>selenide</artifactId> <version>7.0.0</version> <scope>test</scope> </dependency>

Gradle Dependency

// Selenide testImplementation "com.codeborne:selenide:7.0.0"

Initializing Selenide

Unlike Selenium which requires extensive setup, Selenide initializes automatically. However, for customization, you can use the Configuration class:

import com.codeborne.selenide.Configuration; public class BaseTest { static { // Configure browser Configuration.browser = "chrome"; // Base URL for all tests Configuration.baseUrl = "https://demo.example.com"; // Timeout for element to appear Configuration.timeout = 3000; // 3 seconds // Hold browser open after tests Configuration.holdBrowserOpen = false; // Base directory for downloads Configuration.downloadsFolder = "build/downloads"; } }

Basic Operations

Selenide provides intuitive methods for common browser operations. The syntax is clean and readable.

Opening Pages

// Open a URL open("/login"); // Open with full URL open("https://demo.example.com/login"); // Open with authentication open("/admin", admin, password123");

Finding Elements

// Single element - most common selectors $("#username").setValue("testuser"); $("[name='email']").setValue("test@example.com"); $(".btn-primary").click(); // Using text content $("=Submit").click(); $("*=sign in").click(); // Multiple elements $$(".menu-item").first().click(); $$("table tr").get(1).click(); // Filter elements $$(".product").filter(".in-stock").first().click();

Working with Forms

// Text input fields $("#username").setValue("johndoe"); $("#username").append("123"); // Append text $("#search").pressEnter(); // Clear field $("#username").clear(); // Dropdown (select) $("#country").selectOption("United States"); $("#country").selectOptionByValue("US"); $("#country").selectOptionByIndex(1); // Checkbox $("#agree").click(); $("#remember").setSelected(true); // Radio buttons $("[name='payment']").selectRadio("credit_card");

Assertions and Verification

// Text assertions $(".title").shouldHave(text("Welcome, John!")); $(".title").shouldHave(exactText("Welcome, John!")); $(".description").shouldHave(textCaseSensitive("HELLO WORLD")); // Value assertions $("#username").shouldHave(value("johndoe")); $("#email").shouldBe(empty()); // Visibility assertions $(".success-message").shouldBe(visible()); $(".error-message").shouldNotBe(visible()); // Enabled/disabled assertions $("#submit-btn").shouldBe(enabled()); $("#delete-btn").shouldBe(disabled()); // Attribute assertions $("#avatar").shouldHave(attr("alt", "User Avatar")); $("#link").shouldHave(href("https://example.com"));

Waiting Strategies

One of Selenide's strongest features is its automatic waiting. Understanding how it works helps you write more reliable tests.

Default Automatic Waiting

Selenide automatically waits for conditions before performing actions. This eliminates the need for explicit waits:

// No explicit wait needed - Selenide waits up to timeout $("#submit").click(); // Wait for element to appear $(".loading").shouldBe(visible); // Wait for element to disappear $(".loading").shouldDisappear(); // Wait for text to appear $("#status").shouldHave(text("Success"));

Custom Waiting Conditions

import com.codeborne.selenide.Condition.*; import com.codeborne.selenide.ex.*; // Wait with custom timeout $("#result"). should(appear, java.time.Duration.ofSeconds(10)); // Wait for condition with custom message $("#data"). shouldHave(text("Loaded"), Duration.ofSeconds(15)); // Wait until condition is met $("#counter").shouldHave(text("10"));

Page Objects with Selenide

For maintainable test code, use the Page Object pattern with Selenide's elegant syntax:

import com.codeborne.selenide.SelenideElement; import static com.codeborne.selenide.Selectors.*; public class LoginPage { private final SelenideElement usernameInput = $("#username"); private final SelenideElement passwordInput = $("#password"); private final SelenideElement loginButton = $("button[type='submit']"); private final SelenideElement errorMessage = $(".error-message"); private final SelenideElement successMessage = $(".success-message"); public LoginPage open() { open("/login"); return this; } public LoginPage enterUsername(String username) { usernameInput.setValue(username); return this; } public LoginPage enterPassword(String password) { passwordInput.setValue(password); return this; } public DashboardPage clickLogin() { loginButton.click(); return new DashboardPage(); } public LoginPage login(String username, String password) { return enterUsername(username) .enterPassword(password) .clickLogin(); } public String getErrorMessage() { return errorMessage.text(); } public LoginPage shouldShowError(String expectedError) { errorMessage.shouldBe(visible) .shouldHave(text(expectedError)); return this; } }

Using the Page Object

import org.junit.jupiter.api.Test; class LoginTest { @Test void successfulLogin() { new LoginPage() .open() .login("admin", "password123"); // Verify redirect to dashboard $(".dashboard").shouldBe(visible()); } @Test void invalidCredentials() { new LoginPage() .open() .login("admin", "wrongpass") .shouldShowError("Invalid credentials"); } }

// Chrome options example Configuration.browserCapabilities = new ChromeOptions() .addArguments("--start-maximized") .addArguments("--disable-infobars") .addArguments("--disable-extensions") .addArguments("--no-sandbox") .setExperimentalOption("excludeSwitches", Arrays.asList("enable-automation"));

Handling Ajax and Dynamic Content

Selenide excels at handling Ajax-heavy applications where content loads dynamically:

// Wait for Ajax requests to complete $("#load-more").click(); // Wait for element to appear after Ajax call $("#results .item").shouldHave(sizeGreaterThan(0)); // Wait for specific text $("#status").waitUntil(text("Complete"), 5000); // Wait for element to disappear $(".spinner").waitUntil(not(visible()), 3000); // Polling interval for custom checks $("#data").should(until(el -> el.getText().contains("Ready"));

Best Practices

✅

Writing Maintainable Selenide Tests

Follow these practices to create robust, maintainable test suites:

  • Use Page Objects - Encapsulate page logic in dedicated classes
  • Leverage Automatic Waiting - Trust Selenide's built-in waiting
  • Keep Selectors Stable - Use IDs and data-testid attributes
  • Chain Methods - Use fluent API for readable code
  • Handle Asynchrony - Use shouldHave, shouldBe for assertions
  • Organize Test Data - Separate test data from test logic

Key Points

  • Selenide wraps Selenium WebDriver for simpler, more reliable browser automation
  • Automatic waiting eliminates flaky tests caused by timing issues
  • Fluent API makes test code readable and maintainable
  • Built-in screenshots and detailed error messages aid debugging
  • Page Object pattern integrates perfectly with Selenide's syntax
  • Configuration is simple but extensible for complex scenarios
