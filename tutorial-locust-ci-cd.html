CI/CD Integration for Performance Testing

Integrating load testing into your CI/CD pipeline ensures that performance regressions are caught before they reach production. In this tutorial, we'll explore various ways to automate Locust tests in different CI/CD platforms.

What You'll Learn

  • Set up automated load tests in GitHub Actions
  • Configure GitLab CI pipelines for performance testing
  • Create Jenkins pipelines for continuous performance testing
  • Set up Docker-based test environments
  • Configure alerts and notifications

Why Integrate Load Testing?

  • Early Detection: Catch performance issues in development
  • Consistency: Run the same tests on every commit
  • Regression Prevention: Stop performance degradation
  • Confidence: Ship with confidence knowing performance is tested

GitHub Actions Integration

GitHub Actions provides a powerful way to run load tests on every push or pull request.

Basic GitHub Actions Workflow

# .github/workflows/load-test.yml
name: Performance Testing

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  schedule:
    - cron: '0 2 * * *'  # Nightly tests

env:
  LOCUST_HOST: ${{ secrets.TEST_HOST }}
  LOCUST_USERS: 100
  LOCUST_SPAWN_RATE: 10

jobs:
  load-test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          pip install locust locust[fast] fakeredis

      - name: Run Locust load test
        run: |
          locust -f locustfiles/website_test.py \
            --headless \
            --users ${{ env.LOCUST_USERS }} \
            --spawn-rate ${{ env.LOCUST_SPAWN_RATE }} \
            --run-time 5m \
            --host ${{ env.LOCUST_HOST }} \
            --csv results/loadtest \
            --html results/report.html

      - name: Upload results
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: load-test-results
          path: results/
          retention-days: 30

      - name: Upload HTML report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: locust-report
          path: results/report.html

      - name: Check performance thresholds
        run: |
          # Parse results and check thresholds
          python scripts/check_thresholds.py results/loadtest_stats.csv

Pull Request Performance Comments

# .github/workflows/load-test-pr.yml
name: PR Performance Test

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  load-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Run load test
        run: |
          locust -f locustfile.py \
            --headless \
            --users 50 \
            --spawn-rate 10 \
            --run-time 2m \
            --csv /tmp/results \
            --json
            
      - name: Post results as PR comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const stats = fs.readFileSync('/tmp/results_stats.csv', 'utf8');
            
            const lines = stats.split('\n');
            const header = lines[0].split(',');
            const data = lines[1].split(',');
            
            const metrics = {};
            header.forEach((h, i) => {
              metrics[h] = data[i];
            });
            
            const comment = `
            ## ⚡ Load Test Results
            | Metric | Value |
            |--------|-------|
            | Requests | ${metrics['Request Count'] || 'N/A'} |
            | Failures | ${metrics['Failure Count'] || '0'} |
            | Median | ${metrics['Median Response Time'] || 'N/A'}ms |
            | 95th % | ${metrics['95%ile'] || 'N/A'}ms |
            | 99th % | ${metrics['99%ile'] || 'N/A'}ms |
            
            _Test run on PR #${context.issue.number}_
            `;
            
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: comment
            });

Performance Gate Action

# Example with performance gates
jobs:
  load-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Run load test
        run: |
          locust -f locustfile.py \
            --headless \
            --users 200 \
            --spawn-rate 20 \
            --run-time 5m \
            --csv results \
            --exit-code-on-zero-requests 0
            
      - name: Evaluate performance
        run: |
          python scripts/evaluate_performance.py results/stats.csv
        continue-on-error: true
        
      - name: Fail if thresholds exceeded
        if: failure()
        run: |
          echo "❌ Performance thresholds exceeded!"
          exit 1

GitLab CI/CD Integration

GitLab CI provides excellent support for running load tests as part of your pipeline.

Basic .gitlab-ci.yml

# .gitlab-ci.yml
stages:
  - test
  - performance
  - deploy

variables:
  LOCUST_HOST: "https://staging.example.com"
  LOCUST_USERS: "100"
  LOCUST_DURATION: "5m"

performance_test:
  stage: performance
  image: python:3.11-slim
  before_script:
    - pip install locust locust[fast]
  script:
    - locust -f locustfiles/website.py \
        --headless \
        --users $LOCUST_USERS \
        --spawn-rate 10 \
        --run-time $LOCUST_DURATION \
        --host $LOCUST_HOST \
        --html report.html \
        --csv results
  artifacts:
    paths:
      - report.html
      - results_stats.csv
      - results_stats_history.csv
    expire_in: 30 days
    reports:
      junit: results_results.xml
  only:
    - main
    - merge_requests
    - scheduled

performance_regression:
  stage: performance
  image: python:3.11-slim
  script:
    - pip install locust
    - |
      locust -f locustfiles/api_test.py \
        --headless \
        --users 500 \
        --spawn-rate 50 \
        --run-time 10m \
        --host $LOCUST_HOST \
        --csv results
    - python scripts/check_baseline.py results/stats.csv
  artifacts:
    paths:
      - results/
  only:
    - main
  when: manual

Advanced GitLab CI with Multiple Stages

# Advanced GitLab CI pipeline
stages:
  - test
  - load-test
  - security
  - deploy

variables:
  LOCUST_FILE: "locustfiles/api_test.py"

.load-test-template:
  &load-test-template
  image: locustio/locust:2.20
  before_script:
    - mkdir -p reports
  script:
    - locust -f $LOCUST_FILE
      --headless
      --users $LOCUST_USERS
      --spawn-rate $LOCUST_SPAWN_RATE
      --run-time $LOCUST_DURATION
      --host $LOCUST_HOST
      --html reports/locust-report.html
      --csv reports/locust-data
  artifacts:
    paths:
      - reports/
    expire_in: 7 days

smoke-test:
  <<: *load-test-template
  variables:
    LOCUST_USERS: "20"
    LOCUST_SPAWN_RATE: "5"
    LOCUST_DURATION: "2m"
  stage: load-test
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

full-load-test:
  <<: *load-test-template
  variables:
    LOCUST_USERS: "500"
    LOCUST_SPAWN_RATE: "25"
    LOCUST_DURATION: "15m"
  stage: load-test
  rules:
    - if: '$CI_COMMIT_BRANCH == "main"'
    - if: '$CI_PIPELINE_SOURCE == "schedule"'
  allow_failure: true

stress-test:
  <<: *load-test-template
  variables:
    LOCUST_USERS: "1000"
    LOCUST_SPAWN_RATE: "50"
    LOCUST_DURATION: "30m"
  stage: load-test
  rules:
    - if: '$CI_PIPELINE_SOURCE == "schedule"'
  allow_failure: true

Jenkins Pipeline Integration

Jenkins is widely used in enterprise environments. Here's how to integrate Locust.

Jenkinsfile (Declarative Pipeline)

// Jenkinsfile
pipeline {
    agent any
    
    environment {
        LOCUST_HOST = 'https://staging.example.com'
        LOCUST_USERS = '100'
        LOCUST_SPAWN_RATE = '10'
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Install Dependencies') {
            steps {
                sh 'pip install locust locust[fast]'
            }
        }
        
        stage('Run Load Test') {
            steps {
                sh '''
                    locust -f locustfiles/website.py \
                        --headless \
                        --users $LOCUST_USERS \
                        --spawn-rate $LOCUST_SPAWN_RATE \
                        --run-time 5m \
                        --host $LOCUST_HOST \
                        --html report.html \
                        --csv results \
                        --junit results.xml
                '''
            }
        }
        
        stage('Archive Results') {
            steps {
                archiveArtifacts artifacts: 'report.html,results*.csv', fingerprint: true
                junit 'results.xml'
            }
        }
        
        stage('Performance Gate') {
            steps {
                sh 'python scripts/evaluate_performance.py results/stats.csv'
            }
        }
    }
    
    post {
        always {
            cleanWs()
        }
        success {
            slackSend(color: 'good', message: "Load test passed!")
        }
        failure {
            slackSend(color: 'danger', message: "Load test failed!")
        }
    }
}

Jenkinsfile (Scripted Pipeline)

// Scripted Pipeline with Dynamic Load
node {
    def locustUsers = 100
    def passed = false
    
    stage('Setup') {
        checkout scm
        sh 'pip install locust'
    }
    
    stage('Baseline Test') {
        // Run baseline test first
        sh """
            locust -f locustfile.py \
                --headless \
                --users 50 \
                --spawn-rate 10 \
                --run-time 2m \
                --host $LOCUST_HOST \
                --csv baseline_results
        """
        
        // Read baseline metrics
        def baseline = readFile('baseline_results_stats.csv')
        echo "Baseline: ${baseline}"
    }
    
    stage('Full Load Test') {
        // Progressive load test
        [50, 100, 200, 500].each { users ->
            stage("Load Test: ${users} users") {
                sh """
                    locust -f locustfile.py \
                        --headless \
                        --users ${users} \
                        --spawn-rate 20 \
                        --run-time 5m \
                        --host $LOCUST_HOST \
                        --csv results_${users}
                """
                
                // Check results
                def result = sh(
                    script: "python scripts/check_threshold.py results_${users}_stats.csv",
                    returnStatus: true
                )
                
                if (result != 0) {
                    currentBuild.result = 'UNSTABLE'
                    passed = false
                }
            }
        }
    }
    
    stage('Publish Report') {
        archiveArtifacts artifacts: '*.html,*.csv', fingerprint: true
        publishHTML([
            allowMissing: false,
            alwaysLinkToLastBuild: true,
            keepAll: true,
            reportDir: '.',
            reportFiles: 'report.html',
            reportName: 'Load Test Report'
        ])
    }
}

services:

    ports:

Dockerfile for Custom Locust Image

# Dockerfile
FROM python:3.11-slim

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy locust files
COPY locustfiles/ /mnt/locust/

# Set environment variables
ENV LOCUST_HOST=http://app:8000
ENV PYTHONUNBUFFERED=1

# Run Locust
CMD ["locust", "-f", "/mnt/locust/locustfile.py"]

Notifications and Alerts

Configure notifications to stay informed about test results.

Slack Notifications

import os
import json
from locust import events
import requests

SLACK_WEBHOOK = os.environ.get("SLACK_WEBHOOK_URL")

def send_slack_notification(message, color="good"):
    \"\"\"Send notification to Slack\"\"\"
    if not SLACK_WEBHOOK:
        return
    
    payload = {
        "attachments": [{
            "color": color,
            "text": message,
            "fields": [
                {"title": "Environment", "value": os.environ.get("ENV", "staging")},
                {"title": "Branch", "value": os.environ.get("GIT_BRANCH", "main")}
            ]
        }]
    }
    
    requests.post(SLACK_WEBHOOK, json=payload)

@events.test_start.add_listener
def on_test_start(environment, **kwargs):
    send_slack_notification(
        f"🚀 Load test started!\n"
        f"Users: {environment.runner.target_user_count}\n"
        f"Spawn rate: {environment.runner.spawn_rate}/s",
        color="#36a64f"
    )

@events.test_stop.add_listener
def on_test_stop(environment, **kwargs):
    # Get stats
    stats = environment.stats
    
    message = f"""
📊 Load Test Complete!

Total Requests: {stats.total.num_requests}
Failures: {stats.total.num_failures}
Median: {stats.total.median_response_time}ms
95th %: {stats.total.get_response_time_percentile(0.95)}ms
    \"\"\"
    
    if stats.total.num_failures > 0:
        send_slack_notification(message, color="danger")
    else:
        send_slack_notification(message, color="good")

Email Notifications

import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart
from locust import events

def send_email(subject, body, recipients):
    \"\"\"Send email notification\"\"\"
    msg = MIMEMultipart()
    msg['From'] = 'locust@example.com'
    msg['To'] = ', '.join(recipients)
    msg['Subject'] = subject
    
    msg.attach(MIMEText(body, 'html'))
    
    with smtplib.SMTP('smtp.gmail.com', 587) as server:
        server.starttls()
        server.login(os.environ['EMAIL_USER'], os.environ['EMAIL_PASS'])
        server.sendmail(msg['From'], recipients, msg.as_string())

@events.test_stop.add_listener
def on_test_stop(environment, **kwargs):
    stats = environment.stats
    
    if stats.total.num_failures > 0:
        html = f"""
        
⚠️ Load Test Failed

        
Total Requests: {stats.total.num_requests}

        
Failures: {stats.total.num_failures}

        
Error Rate: {stats.total.fail_ratio*100:.2f}%

        """
        send_email(
            "[ALERT] Load Test Failed",
            html,
            ['team@example.com']
        )

    type: soak
