System Architecture

Understand how LoadMagic.AI works under the hood

High-Level Architecture

LoadMagic.AI employs a modern, cloud-native architecture designed for scalability, reliability, and enterprise-grade security.

Client Layer
API Gateway Layer
Core Services Layer
Data & Storage Layer

1. Client Layer

The client layer provides multiple interfaces for interacting with LoadMagic.AI.

🌐

Web UI (Studio)

Browser-based interface for script creation and management.

React TypeScript Monaco Editor WebSocket
🐞

JMeter Plugin

Desktop plugin for local development and testing.

Java Apache JMeter Swing UI
⚡

REST API

Programmatic access for automation and CI/CD.

OpenAPI JSON JWT Auth
💻

CLI Tool

Command-line interface for power users.

Node.js npm

2. API Gateway Layer

The gateway layer handles authentication, security, and request routing.

🔐

Authentication & Security

  • JWT token validation
  • API key management
  • SSO integration (SAML, OAuth)
  • Role-based access control
🚦

Rate Limiting

  • Per-user throttling
  • API quota management
  • Usage tracking
  • Burst handling
🛣️

Request Routing

  • Load balancing
  • Service discovery
  • Request transformation
  • Health checking

3. Core Services Layer

The heart of LoadMagic.AI where the magic happens.

📂

HAR Processing

Parses and analyzes HTTP traffic captured from browsers.

📜

Script Generation

AI-powered generation of JMeter and Locust scripts.

🔗

Correlation Engine

Automatic extraction and handling of dynamic values.

🤖

AI Agents

George, Rupert, Suzy, Carrie, and Quinn for automation.

📊

Results Visualizer

Interactive dashboards for test result analysis.

👥

User Management

Teams, projects, and collaboration features.

4. Data & Storage Layer

Persistent storage for all application data.

📄

Document Store

MongoDB for flexible document storage.

  • User profiles
  • Project configs
  • Script versions
🧠

AI Model Storage

Vector and ML model storage.

  • Trained models
  • Embeddings
  • Training data
📈

Results Store

S3/GCS for large test result files.

  • Test metrics
  • Log files
  • HAR archives

Click on each layer to learn more about its components:

🖥️ Client Layer - Click to expand

Web UI (Studio): The main interface where you upload HAR files, view results, and manage projects. Built with React and Monaco Editor.

JMeter Plugin: For teams preferring desktop tools, integrates directly with Apache JMeter.

REST API: Full programmatic access for automation and CI/CD integration.

🚪 API Gateway Layer - Click to expand

Authentication: Validates JWT tokens and API keys for every request.

Rate Limiting: Ensures fair usage and prevents abuse.

Routing: Directs requests to appropriate backend services.

⚙️ Core Services Layer - Click to expand

HAR Processing: Parses HTTP traffic and extracts endpoints, headers, and payloads.

Script Generation: Creates optimized JMeter/Locust scripts using AI.

AI Agents: Five specialized agents handle different aspects of testing.

💾 Data & Storage Layer - Click to expand

MongoDB: Stores user data, projects, and configurations.

S3/GCS: Stores large files like test results and HAR archives.

AI Storage: Dedicated storage for ML models and embeddings.
