Distributed Load Testing

When you need to simulate thousands or tens of thousands of concurrent users, a single Locust instance won't be enough. Distributed testing allows you to spread the load across multiple machines, generating massive traffic from a coordinated test cluster.

Understanding the Architecture

Locust supports distributed testing through a master-slave architecture:

  • Master Node: Coordinates the test, collects results, and presents the web UI
  • Worker Nodes: Actually execute the tasks and generate load
  • Communication: Workers communicate with the master via TCP (or gRPC in newer versions)

Setting Up Distributed Testing

Let's set up a distributed Locust environment. First, ensure Locust is installed on all machines:

# Install Locust on all machines
pip install locust

# Or with additional dependencies
pip install locust[fast]

Starting the Master Node

The master node controls the test and provides the web interface:

# Start master on port 5557
locust -f locustfile.py \
    --master \
    --port 5557

Additional master options:

# Master with auto-scaling enabled
locust -f locustfile.py \
    --master \
    --port 5557 \
    --expect-workers 4 \
    --headless \
    -u 10000 \
    -r 100 \
    --run-time 1h \
    --csv /tmp/loadtest

Starting Worker Nodes

Workers connect to the master and execute the load test:

# Start worker (specify master host)
locust -f locustfile.py \
    --worker \
    --master-host 192.168.1.100 \
    --master-port 5557

# Worker with custom tag
locust -f locustfile.py \
    --worker \
    --master-host 192.168.1.100 \
    --master-port 5557 \
    --tags "api"

SSH-Based Cluster Launch

For launching multiple workers via SSH:

# Using locust swarm command
locust -f locustfile.py \
    --swarm \
    --headless \
    -u 5000 \
    -r 100 \
    --run-time 30m \
    --csv results/loadtest \
    --host https://api.example.com

# Launch workers on multiple machines
locust -f locustfile.py \
    --master \
    --expect-workers 8

Pro Tip: Auto-Scaling with Docker

For dynamic load testing, consider using Docker Swarm or Kubernetes with Locust. You can automatically scale worker pods based on CPU usage or queue depth, creating truly elastic load testing infrastructure.

Custom Metrics and Events

Locust provides built-in metrics like request count, response time, and failure rate. But for enterprise testing, you'll often need custom metrics specific to your application.

Using the Event System

Locust's event system allows you to hook into various points of the test lifecycle:

from locust import events, TaskSet, task, between, HttpUser
import time
import json

# Define custom event handlers
def on_request_success(request_type, name, response_time, response_length, **kwargs):
    \"\"\"Called when a request succeeds\"\"\"
    print(f"Success: {name} - {response_time}ms")

def on_request_failure(request_type, name, response_time, exception, **kwargs):
    \"\"\"Called when a request fails\"\"\"
    print(f"Failure: {name} - {exception}")

def on_user_error(user_instance, exception, tb, **kwargs):
    \"\"\"Called when a user encounters an error\"\"\"
    print(f"User error: {exception}")

# Register event handlers
events.request_success.add_listener(on_request_success)
events.request_failure.add_listener(on_request_failure)
events.user_error.add_listener(on_user_error)

# Example: Track custom business metrics
class EcommerceUser(HttpUser):
    wait_time = between(1, 3)
    
    def on_start(self):
        self.cart_total = 0
        self.items_viewed = 0
    
    @task(3)
    def view_product(self):
        product_id = random.randint(1, 1000)
        with self.client.get(f"/api/products/{product_id}", 
                           catch_response=True) as response:
            if response.status_code == 200:
                self.items_viewed += 1
                response.success()
            else:
                response.failure(f"Got status {response.status_code}")
    
    @task(2)
    def add_to_cart(self):
        product_id = random.randint(1, 1000)
        with self.client.post("/api/cart/add",
                            json={"product_id": product_id, "quantity": 1},
                            catch_response=True) as response:
            if response.status_code == 200:
                data = response.json()
                self.cart_total = data.get("total", 0)
                response.success()
            else:
                response.failure(f"Failed to add to cart")
    
    @task(1)
    def checkout(self):
        with self.client.post("/api/checkout",
                            json={"payment_method": "card"},
                            catch_response=True) as response:
            if response.status_code == 200:
                # Custom metric: Track checkout conversion
                events.request_success.fire(
                    request_type="CHECKOUT",
                    name="checkout_conversion",
                    response_time=response.elapsed.total_seconds() * 1000,
                    response_length=len(response.content)
                )
                self.cart_total = 0
                response.success()
            else:
                response.failure(f"Checkout failed")

Publishing Custom Metrics

You can publish custom metrics to external monitoring systems:

import statsd
from locust import events

# Initialize StatsD client
statsd_client = statsd.StatsClient('localhost', 8125)

def publish_custom_metrics(request_type, name, response_time, response_length, **kwargs):
    \"\"\"Publish custom metrics to StatsD/Graphite\"\"\"
    # Send response time as timing
    statsd_client.timing(f"locust.{name}.response_time", response_time)
    
    # Send request count
    statsd_client.incr(f"locust.{name}.requests")
    
    # Track custom business metrics
    if "checkout" in name:
        statsd_client.incr("locust.checkout.conversions")
    elif "cart" in name:
        statsd_client.incr("locust.cart.additions")

events.request_success.add_listener(publish_custom_metrics)

# Example: Prometheus metrics export
from prometheus_client import Counter, Histogram, Gauge

REQUEST_COUNT = Counter('locust_requests_total', 'Total requests', ['method', 'endpoint', 'status'])
REQUEST_LATENCY = Histogram('locust_request_latency_seconds', 'Request latency', ['method', 'endpoint'])
ACTIVE_USERS = Gauge('locust_active_users', 'Number of active users')

def export_to_prometheus(request_type, name, response_time, response_length, **kwargs):
    \"\"\"Export metrics to Prometheus\"\"\"
    status = "success"
    REQUEST_COUNT.labels(method=request_type, endpoint=name, status=status).inc()
    REQUEST_LATENCY.labels(method=request_type, endpoint=name).observe(response_time / 1000)

events.request_success.add_listener(export_to_prometheus)

# Track active users
@events.init_command_line_parser.add_listener
def update_active_users(parser):
    parser.add_argument("--active-users", type=int, default=0)

@events.user_added.add_listener

    ACTIVE_USERS.inc()

@events.user_stopped.add_listener
def on_user_stopped(user, **kwargs):
    ACTIVE_USERS.dec()

Data-Driven Testing with CSV

Load test with realistic data from CSV files:

import csv
from locust import HttpUser, task, between, events
import random

# Load test data from CSV
test_data = []

def load_test_data():
    \"\"\"Load test data from CSV file\"\"\"
    global test_data
    with open('test_data/users.csv', 'r') as f:
        reader = csv.DictReader(f)
        test_data = list(reader)

# Load data on startup
load_test_data()

class DataDrivenUser(HttpUser):
    wait_time = between(1, 3)
    
    def on_start(self):
        # Assign random user from test data
        self.user_data = random.choice(test_data)
        self.token = None
    
    @task
    def login(self):
        response = self.client.post("/api/auth/login", json={
            "username": self.user_data['username'],
            "password": self.user_data['password']
        })
        if response.status_code == 200:
            self.token = response.json()["token"]
    
    @task
    def get_profile(self):
        if self.token:
            self.client.get("/api/user/profile",
                         headers={"Authorization": f"Bearer {self.token}"})

Advanced Task Patterns

Creating realistic load patterns requires sophisticated task design. Let's explore advanced patterns for simulating complex user behavior.

Sequential Task Flows

Model realistic user journeys through sequential task execution:

from locust import TaskSet, task, between, HttpUser

class UserJourney(TaskSet):
    \"\"\"Sequential user journey through an e-commerce flow\"\"\"
    
    def on_start(self):
        \"\"\"Initialize user session\"\"\"
        self.cart = []
        self.auth_token = None
        self.user_id = None
    
    @task
    def complete_purchase_flow(self):
        \"\"\"Execute the full purchase journey as a single task\"\"\"
        # Step 1: Login
        self._login()
        
        # Step 2: Browse products
        self._browse_products()
        
        # Step 3: View product details
        self._view_product_details()
        
        # Step 4: Add to cart
        self._add_to_cart()
        
        # Step 5: Update quantity
        self._update_cart()
        
        # Step 6: Checkout
        self._checkout()
    
    def _login(self):
        with self.client.post("/api/auth/login", json={
            "email": "test@example.com",
            "password": "testpass123"
        }, catch_response=True) as response:
            if response.status_code == 200:
                self.auth_token = response.json()["token"]
                response.success()
            else:
                response.failure("Login failed")
    
    def _browse_products(self):
        # Browse category pages
        categories = ["electronics", "clothing", "home"]
        for category in categories[:random.randint(1, 3)]:
            self.client.get(f"/api/products?category={category}")
    
    def _view_product_details(self):
        # View 2-5 random products
        for _ in range(random.randint(2, 5)):
            product_id = random.randint(1, 1000)
            self.client.get(f"/api/products/{product_id}")
    
    def _add_to_cart(self):
        # Add 1-4 items to cart
        for _ in range(random.randint(1, 4)):
            product_id = random.randint(1, 1000)
            self.client.post("/api/cart", json={
                "product_id": product_id,
                "quantity": random.randint(1, 3)
            })
    
    def _update_cart(self):
        # Sometimes update quantities
        if random.random() > 0.5:
            self.client.put("/api/cart/update", json={
                "item_id": random.randint(1, 100),
                "quantity": random.randint(1, 5)
            })
    
    def _checkout(self):
        # Complete checkout
        with self.client.post("/api/checkout", json={
            "address_id": random.randint(1, 100),
            "payment_method": "credit_card"
        }, catch_response=True) as response:
            if response.status_code in [200, 201]:
                response.success()
            elif response.status_code == 400:
                # Out of stock - acceptable
                response.success()
            else:
                response.failure(f"Checkout failed: {response.status_code}")

class EcommerceUser(HttpUser):
    tasks = {UserJourney: 1}
    wait_time = between(5, 15)

Control task frequency using weights:

class WeightedUser(HttpUser):
    wait_time = between(1, 3)
    
    @task(10)
    def browse_homepage(self):
        \"\"\"Most common - weight 10\"\"\"
        self.client.get("/")

    def search_products(self):
        \"\"\"Medium frequency - weight 5\"\"\"
        query = random.choice(["laptop", "phone", "tablet"])
        self.client.get(f"/api/search?q={query}")
    
    @task(3)
    def view_product(self):
        \"\"\"Less common - weight 3\"\"\"
        product_id = random.randint(1, 1000)
        self.client.get(f"/api/products/{product_id}")
    
    @task(1)
    def checkout(self):
        \"\"\"Rare - weight 1\"\"\"
        self.client.post("/api/checkout", json={...})
    
    # Dynamic weight adjustment based on time
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.adjust_weights()
    
    def adjust_weights(self):
        \"\"\"Adjust weights based on time of day\"\"\"
        hour = datetime.now().hour
        if 9 <= hour <= 17:
            # Business hours: more browsing
            self.tasks = {self.browse_homepage: 15, self.checkout: 1}
        else:
            # Off hours: less traffic
            self.tasks = {self.browse_homepage: 5, self.checkout: 1}

Conditional Task Execution

Execute tasks based on conditions:

class ConditionalUser(HttpUser):
    wait_time = between(1, 3)
    
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.is_logged_in = False
        self.has_items_in_cart = False
        self.user_tier = random.choice(["free", "premium", "enterprise"])
    
    @task
    def conditional_login_task(self):
        \"\"\"Execute only if not logged in\"\"\"
        if not self.is_logged_in:
            self.login()
    
    @task
    def conditional_checkout_task(self):
        \"\"\"Execute only if cart has items\"\"\"
        if self.has_items_in_cart:
            self.checkout()
    
    @task
    def tier_based_pricing(self):
        \"\"\"Different behavior based on user tier\"\"\"
        if self.user_tier == "enterprise":
            # Enterprise users get bulk pricing
            self.client.get("/api/products/bulk-pricing")
        elif self.user_tier == "premium":
            # Premium users see deals
            self.client.get("/api/deals/premium")
        else:
            # Free users see standard catalog
            self.client.get("/api/products")

            host="localhost",
            database="testdb",

            password="testpass"
        )
        
        # Redis connection

            host='localhost',
            port=6379,
            db=0

            'postgresql://testuser:testpass@localhost/testdb'
        )
    
    def on_stop(self):
        \"\"\"Close database connections\"\"\"
        self.db_conn.close()
        self.redis_client.close()
    
    @task(3)
    def read_from_database(self):
        \"\"\"Test database read performance\"\"\"
        cursor = self.db_conn.cursor()
        cursor.execute("SELECT * FROM products LIMIT 100")
        results = cursor.fetchall()
        cursor.close()

Database Performance Metrics

import time
from locust import events

def track_database_metrics(db_operation, *args, **kwargs):
    \"\"\"Track database operation metrics\"\"\"
    start_time = time.time()
    
    try:
        # Execute the database operation
        result = db_operation(*args, **kwargs)
        duration = (time.time() - start_time) * 1000  # Convert to ms
        
        # Fire custom event for DB metrics
        events.request_success.fire(
            request_type="DATABASE",
            name=db_operation.__name__,
            response_time=duration,
            response_length=0
        )
        return result
    except Exception as e:
        duration = (time.time() - start_time) * 1000
        events.request_failure.fire(
            request_type="DATABASE",
            name=db_operation.__name__,
            response_time=duration,
            exception=e
        )
        raise

Testing Service Mesh

from locust import HttpUser, task, between, NestedTaskSet
import random

class ServiceUser(HttpUser):
    wait_time = between(0.5, 2)
    
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        # Service URLs
        self.services = {
            "api-gateway": "http://localhost:8000",
            "user-service": "http://localhost:8001",
            "order-service": "http://localhost:8002",
            "payment-service": "http://localhost:8003",
            "notification-service": "http://localhost:8004",
        }
    
    @task(5)
    def api_gateway_request(self):
        \"\"\"Direct API Gateway\"\"\"
        self.client.get(f"{self.services['api-gateway']}/api/v1/products")
    
    @task(3)
    def user_service_chain(self):
        \"\"\"Test user service flow\"\"\"
        # Register -> Login -> Get Profile
        user_data = {
            "email": f"user{random.randint(1,10000)}@test.com",
            "password": "testpass123",
            "name": f"Test User {random.randint(1,1000)}"
        }
        
        # Register
        reg_response = self.client.post(
            f"{self.services['user-service']}/api/users",
            json=user_data
        )
        
        if reg_response.status_code == 201:
            # Login
            login_response = self.client.post(
                f"{self.services['user-service']}/api/auth/login",
                json={"email": user_data["email"], "password": user_data["password"]}
            )
            
            if login_response.status_code == 200:
                token = login_response.json().get("token")
                # Get profile
                self.client.get(
                    f"{self.services['user-service']}/api/users/me",
                    headers={"Authorization": f"Bearer {token}"}
                )
    
    @task(2)
    def order_processing_flow(self):
        \"\"\"End-to-end order processing\"\"\"
        # Create order -> Process payment -> Send notification
        order_data = {
            "items": [
                {"product_id": random.randint(1, 100), "quantity": random.randint(1, 5)}
            ],
            "user_id": random.randint(1, 1000)
        }
        
        # Create order
        order_response = self.client.post(
            f"{self.services['order-service']}/api/orders",
            json=order_data
        )
        
        if order_response.status_code in [200, 201]:
            order_id = order_response.json().get("order_id")
            
            # Process payment
            payment_response = self.client.post(
                f"{self.services['payment-service']}/api/payments",
                json={
                    "order_id": order_id,
                    "amount": random.uniform(10, 1000),
                    "method": "card"
                }
            )
            
            if payment_response.status_code == 200:
                # Trigger notification
                self.client.post(
                    f"{self.services['notification-service']}/api/notifications",
                    json={
                        "order_id": order_id,
                        "type": "order_confirmation",
                        "user_id": order_data["user_id"]
                    }
                )
    
    @task(1)
    def distributed_tracing_test(self):
        \"\"\"Test with distributed tracing headers\"\"\"
        trace_id = f"locust-{random.randint(1,1000000)}"
        
        headers = {
            "X-Request-ID": trace_id,
            "X-B3-TraceId": trace_id,
            "X-B3-SpanId": f"{random.randint(1,1000000):016x}",
        }
        
        response = self.client.get(
            f"{self.services['api-gateway']}/api/v1/dashboard",
            headers=headers
        )

Chaos Engineering with Locust

import random
from locust import events

class ChaosEngine:
    \"\"\"Introduce controlled chaos for resilience testing\"\"\"
    
    def __init__(self, failure_rate=0.1, latency_multiplier=1.0):
        self.failure_rate = failure_rate
        self.latency_multiplier = latency_multiplier
        self.enabled = True
    
    def should_inject_failure(self):
        \"\"\"Determine if failure should be injected\"\"\"
        return self.enabled and random.random() < self.failure_rate
    
    def inject_latency(self, response_time):
        \"\"\"Inject additional latency\"\"\"
        return response_time * self.latency_multiplier

chaos = ChaosEngine(failure_rate=0.05, latency_multiplier=2.0)

@events.request.add_listener
def apply_chaos(request_type, name, response_time, response_length, exception, **kwargs):
    \"\"\"Apply chaos engineering principles\"\"\"
    if exception:
        # Log failure for analysis
        print(f"Chaos injected failure: {name} - {exception}")
    
    # Randomly kill connections to test resilience
    if chaos.should_inject_failure():
        print(f"Chaos: Simulating service failure for {name}")

Performance Threshold Examples

# Define performance thresholds
PERFORMANCE_THRESHOLDS = {
    "api/homepage": {
        "response_time_p95": 500,  # ms
        "error_rate": 0.01,  # 1%
        "throughput": 100,  # rps
    },
    "api/search": {
        "response_time_p95": 1000,
        "error_rate": 0.02,
        "throughput": 50,
    },
    "api/checkout": {
        "response_time_p95": 2000,
        "error_rate": 0.005,
        "throughput": 20,
    },
}

# Validate against thresholds
def validate_performance(results):
    \"\"\"Validate performance against thresholds\"\"\"
    violations = []
    
    for endpoint, metrics in results.items():
        if endpoint in PERFORMANCE_THRESHOLDS:
            threshold = PERFORMANCE_THRESHOLDS[endpoint]
            
            if metrics.get("p95_response_time", 0) > threshold["response_time_p95"]:
                violations.append(
                    f"{endpoint}: P95 response time "
                    f"({metrics['p95_response_time']}ms) exceeds "
                    f"threshold ({threshold['response_time_p95']}ms)"
                )
            
            if metrics.get("error_rate", 0) > threshold["error_rate"]:
                violations.append(
                    f"{endpoint}: Error rate "
                    f"({metrics['error_rate']:.2%}) exceeds "
                    f"threshold ({threshold['error_rate']:.2%})"
                )
    
    if violations:
        raise AssertionError(f"Performance violations:\n" + "\n".join(violations))
